# Contributing

Thank you for contributing to Regbot.

1. Open an issue for significant behavior or public-interface changes.
2. Keep source code, comments, errors, tests, and documentation in English.
3. Add tests for every behavior change.
4. Run `make check` before opening a pull request.
5. Keep destructive behavior fail-closed and explainable.

Commits should be focused and use clear imperative subjects. Pull requests must
describe safety implications and compatibility effects.

By contributing, you agree that your contributions are licensed under Apache
License 2.0.
